Project No. 1
CLP/PLP Redesign
Features
  • More interactive filtering system
  • Submenu displayed at the top of the page
  • Larger product images
  • Wider page width
  • Infinite scroll
Hypothesis
If these features are added to both the category and product pages, the user will be more likely to stay on the site longer and consequently make a purchase.
Process & Challeges
This project was implemented using Adobe Target, meaning the code was being deploying after the browser had loaded and in conjunction with already existing production code. This contributed to one of the major challenges whilst developing this project, which was making sure all of the DOM changes being made weren’t apparent to the user. I made use of the setTimeout() method as well as mutation observer to ensure a smooth user experience.

Project No. 2
Shipping Threshold
Features
  • Animated progress bar
  • Must appear in cart modal, cart page, and on PLP
Hypothesis
If this feature is added, the user will be more likely to add more items to their cart in order to fufill the threshold.
Process & Challeges
The dynamic cart does not exist on the mobile DressBarn site. To subvert that obstacle, I used AJAX to pull the cart subtotal from the desktop cart on every ‘Add to bag’ button click. I used that variable updated as a condition for the progress bar animation to run.

Project No. 2
Positive Reviews
Features
  • Display only five star reviews
  • Reviews must be of an adequete length, not too long
Hypothesis
If these features are added and displayed at the top of the CLP the user will feel more confident and their choice and therefore more likely to add to bag.
Process
The reviews were being displayed via a third party application, so a condition had to be created to wait for the application to appear in the DOM before the reviews would be parsed. At that point, the first five-star review would be displayed in a container at the top of the CLP. Only a specific word could would appear before the rest of the text would be replaced with an ellipses to take the possibility of longer reviews into account.
Project No. 1
Impulse Shade
Features
  • Dynamically present three trending products
  • Fuctional 'Add to bag' button
Hypothesis
If three trending products are presented to the customer on the mobile cart page, the customer will be more likely to add these items to their cart.
Process & Challeges
The trending items were handpicked by the brands team, and so they were pulled using AJAX from their respective product pages. From there, I used another AJAX call so that the ‘Add to bag’ in the impulse shade would correspond with the one on the product page so that it would be functional. Once an item is added to bag, it would be removed from the impulse shade. If the user adds all three items, the impulse shade would disappear.
